Doubble Mobile App
Overview
Doubble is an Investment platform that allows customers of Sterling Bank to easily make investment In Dollars & Naira. The platform was first developed a web application and after request for customer, the Mobile Application project was approved.

My role was to design the User Interface for Android and deliver a Design System that will aid the application development. I collaborated with the Doubble Team to understand the problem and goal of the project. I worked with the Project Managers and a single developer to review their existing web application and ask questions that will inform the design.

The Process

Due to time constraints, I started off by asking questions and performing one-on-one interviews with internal and external users. I reviewed the analytics of the web application to gather feedback that will for my user journeys and to identify possible pain points.

The results boosted the data already gathered to frame the goal and the user's need for the application. I preformed Heuristic Analysis to check available applications within the Nigerian market and reviewed the applications for the following Nielsen Norman Heuristic:

1. Match Between System and the Real World.
2. Consistency and Standards
3. Help and Documentation

With the data gathered I began Sketching the ideas and developed wireframes to communicate my thoughts with the stakeholders. Using wireframes sketched on paper and pencil, I was effective to quickly iterate through page architecture ideas. This helped me to eliminated screens that were unnecessary and merge function into one screen that would serve the purpose better.

Sketch was the best tool for the UI design stage for designing Hi-Fidelity Wireframes. For Prototyping, Zeplin was my choice tool because it provided a Design System that will allow the developer to understand all the design elements.

Summary

The Development of the application is currently on-going and due to be release November 2019.

Problem Statement
Lending in Nigeria is characterized as a tedious process for customers and risky for lenders. Customers complaint revolves around the experience during loan applications due to the tacky structure within organizations.

Over the years, the challenge has loomed, organizations record profit decline annually and the customer recounts a poor experience.
The solution
Specta provided the opportunity of lending to customers a 5 minutes experience eliminating human factor delays and automating the process end to end without stress and hassles of documentation.
Roles & Responsibility
I led the initial design of Spectra for versions 1 & 2 from conceptualization to implementation from October 2017 - March 2019.
